Structured Query Language

SQL

язык структурированных запросов, используемый для управления и работы с реляционными базами данных

SQL был разработан в 1970-х годах в IBM Эдгаром Коддом

История и эволюция SQL

  1. 1970
    Эдгар Кодд предлагает реляционную модель данных. Начинается разработка SQL в IBM
  2. 1986
    SQL стандартизирован Американским национальным институтом стандартов (ANSI) и Международной организацией по стандартизации (ISO)
  3. 1990+
    SQL постоянно развивается и дополняется новыми функциями, такими как поддержка триггеров, хранимых процедур, полнотекстового поиска и других возможностей

Роль SQL в работе с реляционными базами данных

  • Управление данными - позволяет вставлять, изменять и удалять данные в таблицах
  • Управление структурой данных - используется для создания и изменения структур баз данных (таблицы, индексы и представления)
  • Управление данными - возможности для настройки прав доступа, защиты данных и управления транзакциями
  • Управление данными - позволяет фильтровать, сортировать и объединять данные из различных таблиц

Основные задачи SQL

  • Создание и модификация таблиц
  • Извлечение данных
  • Управление целостностью данных

Синтаксис операторов SQL

Data Definition Language (DDL)

используются для определения и изменения структуры базы данных, такой как создание, изменение и удаление таблиц и других объектов

Синтаксис операторов SQL

CREATE TABLE

Создание новой таблицы

CREATE TABLE employees (
	...
);

Синтаксис операторов SQL

ALTER TABLE

Изменение структуры существующей таблицы, добавление или удаление столбцов

ALTER TABLE employees 
ADD COLUMN salary DECIMAL(10, 2);

Синтаксис операторов SQL

DROP TABLE

Удаление таблицы из базы данных
DROP TABLE employees;

Синтаксис операторов SQL

Data Manipulation Language (DML)

операторы DML позволяют изменять данные в таблицах (вставка, изменение, удаление)

Синтаксис операторов SQL

SELECT

Извлечение данных из таблицы

						SELECT first_name, last_name 
						FROM employees 
						WHERE department = 'IT';

Синтаксис операторов SQL

INSERT INTO

Добавление новых данных в таблицу

INSERT INTO employees (first_name, last_name, department) 
VALUES ('John', 'Doe', 'HR');

Синтаксис операторов SQL

UPDATE

Изменение данных в существующих записях

UPDATE employees
SET department = 'Sales' WHERE id = 1;

Синтаксис операторов SQL

DELETE

Удаление данных из таблицы

DELETE FROM employees
WHERE id = 1;

Синтаксис операторов SQL

Data Control Language (DCL)

используется для управления правами доступа к данным

Синтаксис операторов SQL

GRANT

Предоставление прав пользователям на выполнение определенных действий

GRANT SELECT ON employees
TO 'user_name';

Синтаксис операторов SQL

REVOKE

Отзыв ранее предоставленных прав доступа

REVOKE SELECT ON employees
FROM 'user_name';

Синтаксис операторов SQL

Transaction Control Language (TCL)

используются для управления транзакциями, позволяя группировать несколько операций в одну транзакцию, которую можно подтвердить или отменить

Синтаксис операторов SQL

COMMIT

Подтверждение транзакции, запись изменений в базу данных
COMMIT;

Синтаксис операторов SQL

ROLLBACK

Откат всех изменений, выполненных в транзакции, до последнего сохранения
ROLLBACK;
